# BahasaBot

**An AI-powered Bahasa Melayu learning platform for international students.**

BahasaBot is a Final Year Project (FYP) and production-grade full-stack AI application built to help international students learn Bahasa Melayu in context — not through simple word-for-word translation. It combines a RAG-powered AI tutor, structured course generation, adaptive quizzes, a personalized learning roadmap, gamification, and a comprehensive dashboard into a single, cohesive platform.

> **Live Demo:** [https://bahasa-bot.vercel.app](https://bahasa-bot.vercel.app/login)
>
> ⚠️ The backend is hosted on Render's free tier. The **first request after inactivity may take up to 50 seconds** as the server cold-starts. Subsequent requests are fast. Please be patient on first load.

## Overview

BahasaBot helps international students learn Malay through:

- **AI Tutor Chatbot** — contextual Malay teaching via LangChain + Google Gemini + RAG, with streaming SSE responses and session memory
- **Dynamic Course Generator** — user enters any topic and gets a fully structured Topic → Modules → Classes learning path
- **Module Quizzes** — auto-generated after each module; 70% pass threshold required to unlock the next module
- **Adaptive Standalone Quiz** — personalized questions generated from identified weak points and learning history
- **Dashboard** — progress tracking, vocabulary, grammar, weak points, quiz history, leaderboard, and BPS level
- **My Journey** — a personalized course-based learning roadmap generated by Gemini, with deadline tracking, overdue extension, and BPS-upgrade regeneration
- **Spelling Practice Game** — gamified audio-led spelling using the user's own learned vocabulary
- **Gamification** — daily streaks, XP accumulation, milestone notifications, and a weekly XP leaderboard
- **Pronunciation Audio** — one-click Malay word pronunciation via browser-native Web Speech API
- **In-App Notifications** — streak milestones, XP milestones, BPS level-up, journey reminders, course completion
- **Admin Panel** — user analytics, feedback management, journey oversight, leaderboard, system stats
- **Evaluation Feedback** — built-in 3-question survey for FYP usability evaluation evidence
- **Daily Language Tips** — Gemini-generated Malay tips delivered as a daily toast notification
- **Onboarding Flow** — 8-step new-user questionnaire feeding personalization across chatbot, journey, and roadmap generation
- **Profile & Settings** — profile editing, password management, account deletion, feedback submission

## Key Features

### 1. Authentication & User Management
- Email/password registration and login
- Google OAuth login (one-click)
- JWT-based protected API access with refresh token flow
- Mandatory password setup for first-time Google-only accounts (`SetPasswordModal`)
- Forgot password via 6-digit email verification code (Resend)
- Account deletion with identity confirmation

### 2. AI Chatbot (Personal Tutor)
- Conversational Malay tutoring in English or Malay
- Google Gemini + LangChain orchestration with RAG pipeline
- Session-based conversation memory (last 10 messages from PostgreSQL)
- Vocabulary and grammar auto-extracted and saved after each response
- Streaming responses via Server-Sent Events (SSE)
- User profile injected into system prompt for personalized tone and complexity
- RAG context Redis-cached (5 min TTL) — saves ~1.3s on repeat topic queries
- Profile Redis-cached (5 min TTL), history cache updated in-place (no DB re-read per turn)
- App-aware: redirects users to dedicated features (Courses, Quiz, Dashboard, Journey, Games) instead of simulating them

### 3. Dynamic Course Generator
- User enters any topic in English
- Two-pass content filter: regex blocklist + Gemini moderation (ethical validation)
- Gemini generates: course title/objectives → modules → classes with vocab and examples
- Parallel class content generation (asyncio.gather, Semaphore-limited)
- **Course deduplication system**: first generation becomes a reusable template; subsequent users for the same topic get an instant deep clone — no redundant Gemini calls
- Background generation via FastAPI `BackgroundTasks` + Redis job state + floating progress card
- Sequential module locking — must pass quiz to unlock next module

### 4. Quiz System

#### Module Quiz
- Auto-generated after all classes in a module are completed
- 10 questions: 6 MCQ + 4 fill-in-the-blank
- 70% pass threshold required to unlock the next module
- Wrong answers update weak points; Redis-cached (2h TTL)

#### Adaptive Standalone Quiz
- 15 questions: 6 MCQ + 6 fill-in-the-blank + 3 translation
- Personalized from user's accumulated weak points across all courses and chatbot sessions
- Recalculates BPS proficiency level after every attempt
- Full results page: score ring, BPS update banner, per-question breakdown with pronunciation

### 5. Dashboard
- 5-tab layout: Overview / Vocabulary / Grammar / Quiz History / Leaderboard
- Overview redesign: 3 SVG progress rings (Learning / Vocabulary / Quiz), BPS strip, inline emoji stats, quick-action pills
- Weekly XP leaderboard: top-10 users by current-week XP, current-user rank highlighted
- All data refreshes on window focus (30s throttle)

### 6. My Journey — Learning Roadmap
- 3-question setup: intent, learning goal, target timeline (1–6 months)
- Gemini generates a flat course-obstacle roadmap proportional to timeline
- Road/path visual UI: completed (green checkmark), current (glowing pulse), locked (grey + lock icon)
- Overdue detection, one-time deadline extension (1–3 months)
- BPS upgrade banner: regenerate only uncompleted elements after levelling up
- Identity-verified deletion (password for email users, email confirmation for Google users)
- Past journeys section showing completed/deleted roadmaps
- Gemini-generated personalized banner image on creation

### 7. Spelling Practice Game
- Audio-led spelling using the user's own `vocabulary_learned` data
- Leitner-box-inspired word selection (wrong words prioritized ×3)
- 10-second per-word countdown timer with color shift (green → yellow → red pulse)
- Levenshtein fuzzy matching for "Almost!" feedback
- Combo multiplier (×1 → ×1.5 → ×2), session summary with accuracy and XP earned

### 8. Gamification
- Daily streak tracking (consecutive days with any learning activity)
- XP awards: class completed = 10 XP, quiz passed = 25 XP, chatbot session = 5 XP, spelling correct = 2 XP, roadmap obstacle = 100 XP bonus, roadmap completed on time = 500 XP
- All XP awards logged to `xp_logs` table for weekly leaderboard aggregation
- Milestone notifications at 3/7/14/30-day streaks and every 100 XP
- Milestone cards generated as images (Gemini Image API) and attached to notifications

### 9. Notification System
- Bell icon in sidebar with unread badge (React Query polling, deduplicated across instances)
- 5 notification types: `streak_milestone`, `xp_milestone`, `course_complete`, `bps_milestone`, `journey_reminder`
- Clear-all button; panel opens left/right and up/down depending on sidebar state

### 10. Admin Panel
- Stats overview: total users, quiz pass rates, course generation counts, active streaks
- User management: search, paginated table, per-user analytics (recharts), deactivate/delete/reset with password guard
- Feedback management: aggregate stats, star distribution, per-response cards with badge for source
- Journey oversight: read-only table of all user roadmaps with progress, status, and element detail view
- Weekly leaderboard: full table with email, BPS badge, streak, and weekly XP
- Daily Language Tips management: generate new tips by category, toggle active/inactive, browse all

### 11. Evaluation Feedback (FYP)
- 3-question survey shown after every quiz (3s delay): star rating, weak points relevance, open text
- General feedback also accessible from Settings
- Admin feedback page aggregates responses as FYP evaluation evidence

### 12. Daily Language Tips
- Admin generates tips via Gemini by category (grammar, vocabulary, pronunciation, culture)
- One random active tip served daily via Redis-cached endpoint
- Appears as a framer-motion slide-in toast on the dashboard (8s auto-dismiss, sessionStorage dedup)

## Proficiency Framework

BahasaBot uses its own learner-facing proficiency scale — **BahasaBot Proficiency Scale (BPS)**:

| Level | Label | Description |
|---|---|---|
| BPS-1 | Beginner | Core survival vocabulary and basic sentence patterns |
| BPS-2 | Elementary | Common everyday phrases and simple conversations |
| BPS-3 | Intermediate | Extended conversation, reading short texts |
| BPS-4 | Advanced | Complex topics, formal and academic Malay |

CEFR labels (A1/A2/B1/B2) are fully retired across the entire codebase, database, and UI.

## Architecture

```
[User Browser]
      │
[Next.js Frontend — Vercel]
      │
      ├── Auth / session handling (NextAuth.js v5 + JWT)
      ├── UI routes / dashboard / chatbot / journey / games
      │
[FastAPI Backend — Render]
      │
      ├── LangChain orchestration
      ├── Google Gemini API (text + image generation)
      ├── RAG retrieval with pgvector
      ├── Course generation (BackgroundTasks + Redis job state)
      ├── Quiz engine (module + adaptive)
      ├── Journey engine (roadmap + BPS upgrade hooks)
      ├── Gamification / XP / notifications
      │
[PostgreSQL + pgvector — Neon]   [Redis — Redis Cloud]
```

### Key Architectural Decisions

- **Background course generation**: FastAPI `BackgroundTasks` + Redis job state allows the frontend to close the modal immediately and poll progress without blocking.
- **Course deduplication**: First generation per topic slug becomes a template; all subsequent users get a deep clone in milliseconds, eliminating redundant Gemini calls.
- **SSE streaming chatbot**: Responses stream token-by-token via Server-Sent Events with `requestAnimationFrame` batching on the frontend (cuts setState calls from ~100/s to ≤60/s).
- **Redis profile/history cache**: User profile and chat history are cached in Redis and updated in-place — eliminates 2 DB round-trips per chatbot message on the warm path.
- **PgBouncer fix**: `connect_args={"statement_cache_size": 0}` in `create_async_engine()` prevents `prepared statement does not exist` errors from Neon's PgBouncer transaction mode.
- **CORS regex**: `allow_origin_regex=r"https://bahasa-.*\.vercel\.app"` accepts all Vercel preview URLs without hardcoding.

## Database Schema

```sql
-- Core user table (BPS proficiency level stored as BPS-1 through BPS-4)
```

## Security & Reliability

- `bcrypt` password hashing (direct — not via passlib, which has a bug with bcrypt ≥ 4.x)
- JWT access tokens (30 min) + refresh tokens (7 days)
- Google OAuth with mandatory first-login password setup
- CORS locked to frontend domain + regex for Vercel preview URLs
- SQLAlchemy ORM throughout — no raw SQL with user input
- SlowAPI rate limiting: chatbot (20 req/min), course generation (5 req/hr)
- Two-pass ethical content filter on course topic input
- Redis-backed session deduplication (`_sessionInflight` promise in `api.ts`)
- Sentry error monitoring on both frontend and backend
- `statement_cache_size: 0` for Neon PgBouncer transaction-mode compatibility
